# PRD: Browser Polyphonic Synthesizer (Static)

## Summary

A public, static web app: a **polyphonic synthesizer** in the browser using **Web Audio API only** (no audio libraries). Users play notes with the **computer keyboard** and an **on-screen keyboard**, tweak sound with **ADSR, filter, and stereo delay**, view a **realtime waveform**, and **save/load presets** via `localStorage`. No accounts, no backend.

## Goals

- Low-latency, responsive polyphonic playback in modern browsers.
- Familiar **Ableton-style chromatic** computer-key mapping with **octave shift**.
- Self-contained static site suitable for **GitHub Pages**.

## Non-goals (v1)

- MIDI input/output.
- User accounts, cloud sync, or server APIs.
- Sample-based engines, WASM DSP libs, or third-party synth frameworks.
- Mobile-optimized touch performance (best-effort only).
- Recording/export to WAV.

## Users & use cases

- Musicians and developers who want a quick browser instrument without installing software.
- Tinkering with oscillator waveforms, envelopes, filter, and delay; storing favorite patches locally.

## Functional requirements

### Audio engine

- **Polyphony:** up to **8 simultaneous voices**; when a 9th note starts, **steal** the oldest active voice (or equivalent deterministic stealing).
- **Tuning:** **equal temperament**, **A4 = 440 Hz**.
- **Oscillator:** selectable waveform per voice — **sine, sawtooth, square, triangle** (global setting for new notes; all active voices use current setting on retrigger policy: document as “applies to new notes”; optional: all voices update live — prefer **new notes + optional live update for simplicity**).
- **Amp envelope:** **per-voice ADSR** (Attack, Decay, Sustain, Release) controlling gain after oscillator.
- **Filter (optional use):** low-pass (BiquadFilter), with UI for **cutoff** and **resonance (Q)**.
- **Filter envelope (optional):** when enabled, modulates filter cutoff per voice (ADSR or AD-style subset — implement as ADSR targeting cutoff depth + base cutoff).
- **Stereo delay:** delay with **time**, **feedback**, **wet/dry mix**; routed so output is **stereo** (e.g. ping-pong or dual delay lines — implementation choice, must be audibly stereo).
- **Master volume:** slider on master gain before destination.

### Input & note mapping

- **Computer keyboard — chromatic Ableton-style (one octave row):**
  - White keys: **Z X C V B N M** (and extend with **,** / **.** octave shift context — see below).
  - Black keys: **S D G H J** (sharps between corresponding whites in standard Ableton layout).
  - Additional row keys as needed for full octave: align with user-specified set **Z S X D C V G B H N J M** mapping to consecutive chromatic degrees from a movable base note.
- **Octave shift:** **Comma (,)** down one octave, **Period (.)** up one octave (adjust base MIDI note); show current octave/base on UI.
- **Virtual keyboard:** bottom of screen; **click/tap** to trigger notes; **highlight** keys while held (keyboard + mouse).
- **Prevent default** browser behavior for mapped keys when app focused (e.g. space scroll) where applicable.

### UI (dark theme)

- Layout: **virtual keyboard** anchored **bottom**; control panel above.
- Controls (sliders or knobs — consistent control style):
  - Amp: **A, D, S, R**
  - Filter: **cutoff, resonance**; toggle or amount for **filter envelope** + envelope params if enabled
  - Delay: **time, feedback, mix**
  - Oscillator waveform selector
  - Master volume
- **Waveform display:** realtime visualization from **AnalyserNode** (time-domain waveform).
- Short **“how to play”** hint (keyboard map + octave keys).

### Presets

- **Save preset:** name + serialize all relevant parameters to **localStorage**.
- **Load preset:** restore parameters; list/delete presets optional but recommended (minimum: load last / named list).
- **Default preset** on first visit (sensible init patch).

### Public / static

- No authentication; fully client-side.
- Works from `file://` or GitHub Pages base path — use Vite `base` config for **GitHub Pages** project or user site as needed.

## Non-functional requirements

- Target **Chrome, Firefox, Safari** (latest); graceful message if Web Audio unavailable.
- Avoid audio glitches: use **AudioContext** resume on first user gesture.
- Keep main thread responsive; audio graph updates throttled reasonably for UI drags.

## Data model (client-only)

### Preset (JSON in localStorage)

```json
{
  "id": "uuid-or-slug",
  "name": "string",
  "createdAt": "ISO-8601",
  "params": {
    "waveform": "sawtooth",
    "masterVolume": 0.8,
    "octaveOffset": 0,
    "ampAdsr": { "a": 0.01, "d": 0.2, "s": 0.6, "r": 0.3 },
    "filter": { "enabled": true, "cutoff": 2000, "resonance": 1, "feEnabled": false, "feAdsr": {} },
    "delay": { "time": 0.375, "feedback": 0.4, "mix": 0.25 }
  }
}
```

### Runtime state (in memory)

- Active voices map: noteId → { oscillator, gain, filter nodes, envelope state, start time }.
- AudioContext, master bus, analyser tap, delay subgraph.

## Architecture (high level)

```
UI (keyboard, controls, canvas)
  ↔ state / preset manager (localStorage)
  ↔ VoiceManager (allocation, stealing, note on/off)
  ↔ AudioGraph: Osc → Filter? → Amp ENV → Voice mix → Delay → Master → Analyser → Destination
```

## Acceptance criteria

1. User can play **at least 8 held notes**; 9th triggers voice steal without crashing.
2. **Z/S/X/D/C/V/G/B/H/N/J/M** and black-key mapping produce **chromatic** pitches with correct ET tuning.
3. **,** and **.** shift octave; UI reflects change.
4. Virtual keys trigger sound and **highlight** while active.
5. Amp ADSR audible on each note; filter cutoff/resonance affect timbre; delay stereo audible with time/feedback/mix.
6. Waveform display updates in realtime during play.
7. Preset save/load round-trips all exposed parameters after refresh.
8. `npm run build` output deploys to GitHub Pages and app loads without server.

## Implementation milestones

1. Vite scaffold, dark layout shell, AudioContext bootstrap on gesture.
2. Single voice + polyphony manager + stealing.
3. Keyboard + virtual keyboard + octave shift.
4. Filter + optional filter envelope; delay chain.
5. Analyser waveform canvas.
6. Presets localStorage UI.
7. GitHub Pages workflow / `base` path + README.

## Open decisions (defaults for build)

- **GitHub Pages:** assume **project site** (`/repo-name/`) unless user configures user site — document in README.
- **Voice steal policy:** oldest note (FIFO).
- **Filter envelope:** scales cutoff between min/max Hz from base cutoff + depth.
